# Perseus Project Improvement Plan

**Generated:** 2026-02-25 22:03 GMT+8

## Executive Summary

Perseus is a multi-tenant web platform for managing AI robot agents ("nanobots"). This plan outlines key improvements across platform, robot, and MCP server components.

---

## 1. Platform Improvements

### 1.1 Authentication & Multi-Tenancy
- [ ] **Token Refresh Mechanism**: Implement JWT refresh token flow to replace never-expiring robot tokens
- [ ] **SSO Integration**: Add support for OAuth providers (Google, GitHub)
- [ ] **Session Management**: Add session invalidation and device management
- [ ] **MFA Support**: Implement two-factor authentication for tenant owners

### 1.2 UI/UX Enhancements
- [ ] **Chat Interface Polish**: Improve @mention dropdown with fuzzy search and robot status indicators
- [ ] **Real-time Updates**: Add WebSocket support for instant message delivery (beyond SSE)
- [ ] **Dark Mode**: Implement theme switching capability
- [ ] **Mobile Responsive**: Ensure all pages work on mobile devices
- [ ] **File Preview**: Add inline preview for images, PDFs, and code files in workspace

### 1.3 Robot Management
- [ ] **Robot Health Monitoring**: Dashboard showing robot online/offline status, last heartbeat
- [ ] **Robot Logs Viewer**: In-platform view for robot execution logs
- [ ] **Bulk Operations**: Select and manage multiple robots at once
- [ ] **Robot Templates**: Save and reuse robot configurations

### 1.4 Workspace Enhancements
- [ ] **File Versioning**: Track changes to workspace files over time
- [ ] **Shared Links**: Generate shareable links for workspace files
- [ ] **File Search**: Full-text search across workspace documents
- [ ] **Storage Quotas**: Implement per-tenant/team storage limits

## 2. Robot (Nanobot) Improvements

### 2.1 Platform Connectivity
- [ ] **Connection Health Check**: Automatic reconnection with exponential backoff
- [ ] **Offline Queue**: Queue messages when disconnected, sync when reconnected
- [ ] **Heartbeat Protocol**: Regular ping/pong to report robot status

### 2.2 Tool Integration
- [ ] **More Built-in Tools**: File operations, web search, code execution
- [ ] **Custom Tool Registry**: Allow users to define custom tools
- [ ] **Tool Usage Analytics**: Track which tools are most used

### 2.3 AI Model Support
- [ ] **Model Flexibility**: Support for OpenAI, Anthropic, Ollama, Gemini
- [ ] **Model Fallback**: Automatic failover if primary model fails
- [ ] **Cost Tracking**: Monitor API usage per robot/task
